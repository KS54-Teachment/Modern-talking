# Теоретический блок. Основные понятия Git и GitHub

Краткие определения ключевых терминов, изученных в рамках третьего дня.

| Термин | Определение |
|--------|-------------|
| **Система контроля версий (VCS)** | Программное обеспечение, которое отслеживает изменения файлов во времени, позволяет хранить историю, возвращаться к предыдущим версиям и работать над проектом нескольким людям одновременно. |
| **Git** | Распределённая система контроля версий. У каждого участника есть полная копия истории проекта. Работает локально и не требует постоянного подключения к серверу. |
| **GitHub** | Облачный сервис для хранения Git-репозиториев и совместной работы: Issues, Pull Requests, code review, Wiki, Pages, автоматизация. |
| **Репозиторий (Repository)** | Хранилище проекта со всеми файлами и полной историей изменений. Бывает локальным (на компьютере) и удалённым (на GitHub). |
| **Commit** | Зафиксированный снимок изменений с уникальным идентификатором, автором, датой и комментарием. Базовая «единица истории» в Git. |
| **Branch (ветка)** | Независимая линия разработки. Позволяет вести работу над функцией отдельно, не затрагивая основную ветку (`main`). |
| **Merge (слияние)** | Объединение изменений из одной ветки в другую. Например, слияние `feature-security` в `main`. |
| **Pull Request (PR)** | Запрос на слияние изменений из вашей ветки в основную. Позволяет провести обсуждение и code review перед объединением. |
| **README** | Главный файл-визитка проекта (обычно `README.md`). Описывает, что это за проект, как им пользоваться, кто авторы. |
| **Issues** | Система задач и обсуждений: баги, фичи, поручения. Для задач указывают исполнителя, метки, приоритет, срок и статус. |
| **Wiki** | Встроенная в GitHub база документации проекта в формате вики-страниц, отделённая от основного кода. |

## Базовый рабочий процесс (Git flow)

1. `git clone` — клонировать репозиторий.
2. `git checkout -b feature-…` — создать свою ветку.
3. Внести изменения в файлы.
4. `git add .` → `git commit -m "…"` — зафиксировать изменения.
5. `git push origin feature-…` — отправить ветку на GitHub.
6. Открыть **Pull Request** → code review → **Merge** в `main`.
